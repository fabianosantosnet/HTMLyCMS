Amplie as funcionalidades do CMS

Funções de widgets embutidas no HTMLy

No momento o CMS não tem um gerenciador de plugins (widgets) em seu painel de controle, nesse caso a remoção e inserção de funções é manual direto no tema. No HTMLy você encontra os principais recursos da maioria das plataformas de blog.

  • search($text = null) - função de busca (opcional: informe um nome para o botão)
         Ao criar um formulário certifique-se de definir os atributos name e type para 'search':
         <form><input type="search" name="search" class="form-control" placeholder="Digite para buscar"></form>

  • menu($custom = null) - imprime o menu (parametro opcional: classe CSS);

  • recent_posts($custom = null, $count = null) - por padrão, sem especificar os parametros recent_posts(), retorna a lista de posts recentes pelo título;
         caso especifique o primeiro parâmetro recent_posts(true) retorna um array;
         você também pode informar o número de resultados recent_posts(true,5);

  • popular_posts($custom = null, $count = null) - retorna os posts populares - parâmetros com as mesmas funcionalidades descritas acima;
  • archive_list($custom = null) - mostra a lista de arquivos (padrão false);
  • tag_cloud($custom = null) - lista de tags disponíveis (padrão false);

  • category_list($custom = null) - exibe qualquer categoria que tenha conteúdo;
  • get_related($p->related) - mostra postagens relacionadas por tag exibindo apenas o título da postagem com link para a mesma;
         e para customizar para que retorne um array, especifique o segundo parâmetro get_related($p->related,true);

  • recent_type($type, $custom = null, $count = null) - retorna o postagem recente pelo tipo de conteúdo: video, audio, image (...);
         no exemplo a seguir, retorna um array com 5 itens dos posts do tipo imagem recent_type('image', true, 5);
